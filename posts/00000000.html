becoming a better developer

I'm obsessed with CS, I have been since I was a kid. When I was 12 I had this huge library of (mostly unfinihsed) games that I wasn't really proud of. Fast forward 5 years, I have this huge library of competative programming problems with not much else to show for it. I've spent plenty of time in vim and vscode, but I wouldn't call myself that good of a programmer. I don't think I can use CS to make anything anyone actually cares about right now. This blog is a way of making up for that.

I want to focus on projects that are more substantive. Projects that either A: force me to learn a new subset of theory or new techonolgy or B: have a real practical use for me or other people. Additionally, I want to explore some form of hardware this year.

I want to make 4 projects in 2024, (i'll count this as a project lol). For each project, I want it's scope to be large enough that it'll take me a couple of months to complete. A signifcant portion of these weekly blog posts will probably be me talking about progress on some project. So 0x100 pulls double duty. It satisfies condition A while letting me detail and describe all future projects, long with their goals.

0x100

Before I move on, I'm going to describe in more detail the specific technical goals I have for 0x100. Here's a short summary.

  1. Figure out how the hell i'm supposed to organize web development projects
  2. Understand why react is needed
  3. Practice using bootstrap as intended
  4. Build my own markdown transpiler
  5. Practice writing non-fucked maintable code in general

My web development experience in general has been pretty fucked. It feels like there's a million ways to do anything and everything. Anything I write more complicated than a todo app seems to have a gorillion moving parts, held together by tape and glue. For instance, look at hive, an app I made for my district's Congressional App Challenge. I used react, bootstrap, and firebase to write the whole thing and I barely understood what I was doing.

The only reason I decided to use react and firebase was because that's what "real developers" used. React felt like overkill constantly. I didn't understand the motivation behind any of the architectual choices of the framework, and god forbid you ask me what an effect is. Without firebase I would've been totally fried. There was no way I could've navigated running a backend while also figuring out how to weild the greatsword that is react.

Bootstrap was the only decision that kind of made sense. I stared using Bootstrap after an attempt to use pure CSS resulted in a shitstorm I was not prepared for. Unlike the other two frameworks I used, I felt like I understood why Boostrap existed, and what problems it solved. Don't get me wrong, the way I used Bootstrap was fucked, but I understand why it's fucked.

So this project is a way to address these issues. The only framework I'm using to write 0x100 is Bootstrap 5.3. Everything else is pure HTML and JS, with a couple of automation features (and the markdown transpiler) written in python. I'll be hosting this on github pages (because i'm broke) and working around those limitations. Hopefully the experience is painful enough to motivate the use of react/other frameworks.

Becoming a Better Person

I've been reading a lot of blog style media recently like Astal Codex Ten and LessWrong, which has opened my eyes to the value of good writing. Writing kind of forces you to clarify your thoughts. You're pulling them from the ether of your mind into reality, and once they're out, you can't control how others may percieve them.

I've noticed that I'm incapable for following through on on any sort of long term plan (i.e. 6 months) because even if I write up the plan, I forget the plan exits maybe just two weeks later. Being forced to write a weekly post reflecting on the past week seems like a great way to commit to goals. We'll see how it goes.
